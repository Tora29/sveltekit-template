# コードレビュー

## レビュープロセス

1. コード構造と組織を分析
2. 潜在的なバグやエッジケースをチェック
3. 可読性と保守性の改善を提案
4. プロジェクト規約への準拠を確認

## チェックポイント

### セキュリティ
- 入力検証
- SQLインジェクション対策
- XSS対策
- 認証・認可の適切な実装

### パフォーマンス
- 不要なループや計算
- メモリリーク
- N+1クエリ

### 可読性
- 命名規則
- コメントの適切さ
- 関数の長さ

### テスト
- テストカバレッジ
- エッジケースのテスト

## 出力形式

```markdown
## レビュー結果

### 重要度: 高
- [ファイル:行] 問題の説明

### 重要度: 中
- [ファイル:行] 問題の説明

### 提案
- 改善提案
```
```

## ポイント

1. **簡潔**: 約50行で完結
2. **具体的**: チェックポイントを明示
3. **柔軟**: 高い自由度（コンテキストに応じた判断を許容）
4. **出力形式**: 期待する形式を示す
