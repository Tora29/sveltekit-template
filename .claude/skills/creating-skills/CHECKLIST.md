# スキル品質チェックリスト

スキルを公開する前に確認してください。

## コア品質

- [ ] **name**: 小文字、数字、ハイフンのみ（最大64文字）
- [ ] **name**: 動名詞形を推奨（例: `processing-pdfs`）
- [ ] **name**: 予約語なし（`anthropic`, `claude`）
- [ ] **description**: 空でない（最大1024文字）
- [ ] **description**: 何をするか＋いつ使うかを含む
- [ ] **description**: 三人称で記述
- [ ] **SKILL.md**: 500行以下
- [ ] 追加の詳細は別ファイルに分離
- [ ] ファイル参照は1レベル深さまで
- [ ] 時間に敏感な情報がない
- [ ] 全体で一貫した用語
- [ ] 例は抽象的ではなく具体的

## 構造

- [ ] YAMLフロントマターが正しい形式
- [ ] 参照ファイルへのリンクが正しい
- [ ] ファイルパスはフォワードスラッシュ（`/`）を使用
- [ ] ファイル名は説明的

## コンテンツ

- [ ] Claudeが既に知っていることは省略
- [ ] 適切な自由度を設定
  - 高: テキストベースの指示
  - 中: 疑似コードまたはパラメータ付きスクリプト
  - 低: 特定のスクリプト（変更不可）
- [ ] 複数のオプションを提供しすぎていない
- [ ] デフォルトを明確に推奨

## ワークフロー（該当する場合）

- [ ] チェックリスト形式で進行状況を追跡可能
- [ ] 各ステップが明確に説明されている
- [ ] 検証ステップが含まれている
- [ ] エラー時の対処が記載されている

## コード/スクリプト（該当する場合）

- [ ] スクリプトはエラーを明示的に処理
- [ ] マジックナンバーがない（すべての値が説明されている）
- [ ] 必要なパッケージがリストされている
- [ ] 実行方法が明記されている
- [ ] スクリプトを「読む」か「実行する」か明確

## 避けるべきこと

- [ ] Windowsスタイルのパス（`\`）なし
- [ ] 深くネストした参照なし
- [ ] 曖昧な説明なし
- [ ] 過度な説明なし
- [ ] 時間に敏感な日付なし

## テスト

- [ ] 実際のタスクでテスト済み
- [ ] 意図したモデル（Haiku/Sonnet/Opus）でテスト済み
- [ ] Claudeがファイルを正しくナビゲートできることを確認
